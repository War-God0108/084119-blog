## 起因

上个月搭好了这个博客，写了"你好，世界"，然后就搁置了。

前两天（6 月 14 日），我打开电脑，想着把 Claude Code 的环境好好整一整。当时觉得应该很快——配几个设置而已，能有多难？

结果这一整，就是两天。

## 第一天：我以为我在"配置"，其实我在"调试"

事情的起点很简单：我想让 Claude Code 在任务完成时发出一个提示音，这样我切到别的窗口干活时，不用一直盯着它。

听起来像个十分钟的事情。

Claude 告诉我可以设置 Hook。它帮我写了个 PowerShell 脚本，配置了 `settings.json`。第一个版本很快出来了——但没声音。

然后我们就进入了调试循环：没声音 → 改脚本 → 有声音了但弹窗位置不对 → 改位置 → 弹窗挡住操作了 → 改实现方式 → 切换到别的程序看不到弹窗了 → 换方案 → 声音和弹窗不同步 → 调延迟……

如果你翻看那天的对话记录，大概长这样：

> "没听到"  
> "再来一次"  
> "看到了，但不能操作"  
> "试试"  
> "不行，看到就无法操作"  
> "试试"  
> "行吧也只能这样妥协了"

我数了一下，光是为了调试这个弹窗提示功能，我来回测试了不下三十次。

最终的结果呢？弹窗勉强能用，但有几个妥协：在某些窗口焦点状态下还是会被挡住；声音用了系统提示音而不是自定义音效；hook 的触发延迟无法完全消除。不算完美，但够用了。

**感受一：AI 写代码很快，但让代码在真实环境里跑起来，还是要靠人一遍遍地试。**

## 第一天晚上到第二天：做一些"正经事"

Hook 搞完之后，我开始让 Claude Code 帮我做一些实际的事情。

梳理了我电脑上所有的 CLI 工具——哪些在用、哪些可以删、哪些有冲突。这件事如果让我自己干，大概需要翻遍环境变量和各个目录，但 Claude 帮我在几分钟内完成了。

然后就是你们看到的这个博客。从初始化 Astro 项目，到写文章、配暗色模式、调中文字体——每一步都是我跟 Claude 一起做的。我说想法，它出代码；我看到效果，再让它改。

这种协作模式有一个很有意思的地方：**它降低了"动手"的门槛。** 以前想搭个博客，光是想想要配 Webpack、调 PostCSS、写响应式布局，可能就懒得动了。现在我可以直接说"我想要一个深色背景的文章列表"，它给我代码，我看看效果，不满意就改。

打个比方：以前的编程像是在用笔画画——每一笔都是你自己画出来的，所以你的技法决定了你能画出什么。现在像是在做导演——你负责提要求、做判断、给反馈，AI 负责执行。你的审美和判断力决定了最终作品的质量，但技法的限制大大降低了。

## 它聪明吗？聪明。它傻吗？也傻。

这两天的相处让我对 Claude Code 的能力边界有了更清晰的认识。

**它特别擅长的事情：**
- 写你明确知道要什么的代码——框架代码、配置、脚本，这些对它来说轻车熟路
- 解释你不懂的东西——你可以问"这段配置是什么意思"，它会解释得很清楚
- 帮你踩坑——它知道某个库的某个版本有坑，会提前告诉你

**它特别不擅长的事情：**
- 试错——当你让它"试试另一个方案"时，它有时候会卡在同一种思路上打转。我那天调试弹窗功能时深有体会：它反复用同一种方法但期望不同的结果，最后还是我主动说"换方法"
- 感知真实环境——它不知道弹窗在你的屏幕上是怎样的，不知道声音有没有响。所有涉及物理世界的反馈，它都只能靠你告诉它
- 知道什么时候该放弃——AI 似乎没有"这样做下去没意义了"的判断。它会把一个方向尝试到天荒地老，除非你喊停

这让我想起一个说法：**AI 像一个无限耐心但缺乏判断力的初级工程师。** 你让它干什么它就干什么，从来不抱怨，从来不偷懒——但它也不会在你走错路的时候拉住你。

## 一些意料之外的收获

这两天里，除了完成了一些实际的事情，我还有一些意外的感悟：

**关于"快"与"慢"。** AI 写代码确实快，但决策反而变慢了——因为你有更多选项要评估。以前自己写，方案是有限的，你只能做出自己能力范围内的东西。现在 AI 能给你三个方案，你就要花时间比较。快的是执行，慢的是选择。

**关于挫败感。** 那天调试弹窗的时候，我有几个瞬间真的很烦躁——明明是个小功能，怎么搞了这么久。但后来想想，这跟以前自己 debug 一个诡异 bug 有什么区别？工具变了，但工程中"最后 10% 花 90% 时间"的规律没变。

**关于信任。** 我现在会比较自然地信任 Claude 在一些标准化的任务上——比如写个脚本、配个环境变量。但在涉及系统层面的改动时，我会让它先解释清楚再动手。这种"信任的粒度"是慢慢形成的。

## 我现在怎么看 Claude Code

如果用一句话总结：**它是一个加速器，但方向盘还是在自己手里。**

它不会替你思考、替你判断、替你承担后果。它只是让你从想法到代码的距离变短了。走快点还是走错路，取决于你自己。

有点像开车——以前是走路，每一步都得自己迈；现在有了车，可以开得很快，但路线得自己选，出了事故也得自己负责。

我的建议是：如果你还没用过这类工具，可以试试。但别带着"它能帮我搞定一切"的预期。带着"它是个靠谱的帮手，但我是负责人"的心态，会舒服很多。
